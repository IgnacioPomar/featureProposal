# SSL Application PoC

A **Spring Boot 4 + Java 21** proof of concept that combines:

- HTTPS service startup
- OpenAPI-driven CRUD generation
- Hibernate/JPA persistence
- Liquibase migrations
- CLI setup and installation diagnostics
- Certificate renewal (CLI + HTML page)

## Features

1. HTTPS API on port `8443`.
2. OpenAPI docs at `/api-docs` and Swagger UI at `/swagger-ui.html`.
3. Task CRUD generated from provider OpenAPI spec and connected to service/repository logic.
4. Liquibase changelog-based schema management.
5. Interactive installer (`--setup`) that asks values, validates DB connectivity, generates an installation certificate, applies Liquibase, and saves definitive config.
6. Standalone installation check (`--check-installation`) that does not bootstrap Spring context and always emits an HTML report.
7. Certificate renewal utility via CLI (`--renew-certificate`) and web page (`/certificate-renewal`).

Provider spec used for generation:

- `src/main/openapi/task-crud-provider.yaml`

Generated sources (build output):

- `target/generated-sources/openapi/...`

## Database and Liquibase

- Default DB URL (fallback): `jdbc:h2:file:./target/db/sslapp;MODE=PostgreSQL;DB_CLOSE_DELAY=-1;AUTO_SERVER=TRUE`
- Main changelog: `classpath:db/changelog/db.changelog-master.json`
- Main table: `task_item` (UUID primary key)
- Comparison table: `archive` (YAML example changelog)

`--setup` runs Liquibase update explicitly and prints an **Update Summary** (planned/applied counts + latest applied changesets) so you can track what was done without reading full technical logs.

## Configuration Model

### Standard Spring properties (recommended)

- `server.ssl.key-store-password`
- `spring.datasource.url`
- `spring.datasource.username`
- `spring.datasource.password`
- `app.certificate-page.target-keystore`

Generated by default into:

- `./config/application.properties`

Spring Boot loads `./config/application.properties` automatically on startup.

### Environment placeholders in packaged `application.yml`

- `SSL_KEYSTORE_PASSWORD` (default `changeit`)
- `APP_DATASOURCE_URL`
- `APP_DATASOURCE_USERNAME`
- `APP_DATASOURCE_PASSWORD`
- `APP_CERT_TARGET_KEYSTORE`
- `APP_CERT_ALIAS` (default kept as application alias)

## CLI Commands

### Setup wizard

```bash
./mvnw spring-boot:run -Dspring-boot.run.arguments="--setup"
```

What it does:

- asks SSL and DB values interactively
- warns if external config may override setup values
- validates DB connection
- generates temporary installation certificate (with notice)
- runs Liquibase migration
- saves definitive config file (default `./config/application.properties`)

### Installation check (standalone)

```bash
./mvnw spring-boot:run -Dspring-boot.run.arguments="--check-installation"
```

Optional HTML output file:

```bash
./mvnw spring-boot:run -Dspring-boot.run.arguments="--check-installation --to-file=./target/check-installation-report.html"
```

What is validated:

- required configuration variables/properties
- DB connection
- required tables and required fields from Liquibase changelog
- extra fields reported as warning
- default DB config warning
- H2 warning as volatile database

Report notes include source origin (for example: `./config/application.properties`, `env:...`, `arg:--...`, `classpath:application.yml`).

### Certificate renewal (CLI)

```bash
./mvnw spring-boot:run -Dspring-boot.run.arguments="--renew-certificate --renew-directory=/workspace/testdata/certs/180d-pem --renew-password=changeit"
```

Accepted input channels:

- CLI args: `--renew-directory`, `--renew-password`
- environment: `RENEW_DIRECTORY`, `RENEW_PASSWORD`

## Run and Test

curl -k https://localhost:8443/api-docs
curl -k https://localhost:8443/certificate-renewal

```bash
./mvnw test
```

## Project Structure

- `src/main/java/com/example/ssl/cli/` CLI setup/check/renew utilities
- `src/main/java/com/example/ssl/crud/` entity/repository/service/controllers for CRUD
- `src/main/java/com/example/ssl/web/` web endpoints (`hello`, certificate page)
- `src/main/openapi/` provider OpenAPI YAML
- `src/main/resources/db/changelog/` Liquibase changelogs
- `src/main/resources/application.yml` packaged defaults
